2) Единый глоссарий статусов (для всего проекта)
2.1. Статусы этапов (pipeline / stages)
Код	Текст в UI	Пояснение (tooltip/подпись)
pending	В ожидании	Этап поставлен в очередь
queued	В очереди	Ожидает ресурсов/слота
processing / running	В работе	Выполняется прямо сейчас
completed / success	Готово	Успешно завершён
failed / error	Ошибка	Не удалось выполнить
canceled / cancelled	Отменён	Остановлен вручную/системой
retrying	Повторная попытка	Система пробует снова
paused	Пауза	Приостановлен

В UI всегда показывать одну форму: “Готово”, “В работе”, “Ошибка”… без англ.

3) Таски по компонентам (React/Vue) для IDE

Ниже — разбиение на компоненты и тикеты. Формулировки специально “девелоперские”.

3.1. Общие компоненты (shared)

TICKET A1 — StatusBadge

Компонент бейджа статуса с маппингом code -> {label, tone}

Поддержать группы: order/stage/key/payment

Параметры: type, status, tooltip?

TICKET A2 — Toast/Notifications

Единый сервис уведомлений: success/warn/error

Сообщения берутся из словаря (i18n/const)

TICKET A3 — ConfirmModal

Универсальная модалка подтверждения действий

Props: title, description, confirmLabel, cancelLabel, danger?

TICKET A4 — FormField

Обёртка для label + helper + error

Унификация отступов и стилей

TICKET A5 — MaskedSecretInput

Поле для API-ключа с show/hide, copy, и маской при blur

Поддержать “вставку” и “очистку”

3.2. Страница «Провайдеры / Конфигурация LLM»

TICKET B1 — ProviderKeyForm

Select провайдера

SecretInput для ключа

Save button + loading

Валидация: required, min length

Success toast

TICKET B2 — ModelsForm

Select текстовой модели

Select vision модели (может быть disabled если провайдер без vision)

(optional) Slider “Креативность”

Save button

TICKET B3 — CurrentConfigCard

Read-only карточка текущей конфигурации (без JSON)

Отображение “не выбрано”, если поля пустые

TICKET B4 — KeysHistoryTable

Таблица ключей: masked key, provider, status badge, last check

Actions: Activate / Delete / Check

ConfirmModal для delete/activate

TICKET B5 — API integration layer

Привести ответы API к единой модели во фронте:

ProviderConfig, KeyItem, ModelsConfig

Любые JSON поля парсить внутри, наружу отдавать типизированно

3.3. Страница «Детали заказа»

TICKET C1 — OrderHeader

Заголовок “Заказ №…”, даты, общий статус (StatusBadge)

TICKET C2 — OrderSummaryCard

Сводка: статус, оплата, сумма, провайдер, модель

Человекочитаемые значения из глоссария

TICKET C3 — StagesTable

Таблица/карточки этапов (responsive)

Columns: name, status badge, price, actions

Actions:

RestartStage (confirm)

CancelStage (confirm)

ViewResult (если есть)

ViewError (если failed)

TICKET C4 — StageActions API

Методы: restart/cancel + обработка ошибок

Ошибки приводить к human-readable текстам

TICKET C5 — Hide technical context

Удалить блок Контекст (JSON) из UI полностью

Если есть нужные поля из контекста — вывести отдельными строками (только whitelisted keys)
